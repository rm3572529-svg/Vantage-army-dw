Developer Reference

Server & Deployment Guide

Technical reference for deploying and configuring VantageDown.

Architecture Overview

VantageDown is a fully client-side static site with Firebase as its backend. No Node.js server is required. All logic runs in the browser.

Frontend

HTML5, CSS3, Vanilla JS. No build step required. Static files only.

Authentication

Firebase Auth with Google OAuth and Email/Password providers.

Database

Google Firestore (NoSQL). Collections: users, links, config.

File Storage

No storage needed. Files remain on Google Drive.

Setup Instructions
1
Create a Firebase Project

Go to console.firebase.google.com, create a new project, and enable Firestore Database and Authentication.

2
Configure firebase.js

Open firebase.js and replace the FIREBASE_CONFIG object with your project's config from the Firebase console (Project Settings → Your apps → Web app).

3
Enable Auth Providers

In Firebase Console → Authentication → Sign-in method: enable Google and Email/Password.

4
Set Firestore Security Rules

In Firebase Console → Firestore → Rules, paste the rules from the section below.

5
Create First Admin

Register an account via the site, then go to Firestore → users → your UID document, and manually change role from "user" to "admin".

6
Deploy to a Static Host

Upload all files to any static hosting: Firebase Hosting, Netlify, Vercel, GitHub Pages, or any web server. No backend required.

Firestore Security Rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users: read own doc, write only own doc
    match /users/{uid} {
      allow read:  if request.auth != null && request.auth.uid == uid;
      allow write: if request.auth != null && request.auth.uid == uid;
      // Admin can read/write all users
      allow read, write: if request.auth != null
        && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }

    // Links: authenticated users can create own links, read any link
    match /links/{linkId} {
      allow read: if true; // public read for download pages
      allow create: if request.auth != null;
      allow update, delete: if request.auth != null
        && (resource.data.uid == request.auth.uid
          || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin');
    }

    // Config: public read, admin write only
    match /config/{docId} {
      allow read: if true;
      allow write: if request.auth != null
        && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
  }
}
Recommended Hosting Options
Firebase Hosting

Best option. Free SSL, CDN, same ecosystem as auth/db. Run firebase deploy.

Netlify

Drag and drop the folder. Free tier, auto SSL, instant deploys.

Vercel

Connect your GitHub repo and deploy automatically on each push.

GitHub Pages

Push to a gh-pages branch. Free hosting at username.github.io.

Admin Admin Panel Access

The admin panel is located at /admin.html. It is protected by Firebase role-based access — only accounts with role: "admin" in Firestore can use it. Anyone else sees an "Access Denied" screen.

For additional security, you can rename admin.html to a secret filename (e.g., cp-8f3a92.html) and share only with trusted admins.
